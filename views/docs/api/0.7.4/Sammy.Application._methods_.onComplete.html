Sammy.Application onComplete ( callback )

Comments | Show in GitHub

Adds a onComplete function to the application. onComplete functions are executed at the end of a chain of route callbacks, if they call next(). Unlike after, which is called as soon as the route is complete, onComplete is like a final next() for all routes, and is thus run asynchronously

Example

app.get('/chain',function(context,next){

console.log('chain1');

next(); },function(context,next){

console.log('chain2');

next(); }); app.get('/link',function(context,next){

console.log('link1');

next(); },function(context,next){

console.log('link2');

next(); }); app.onComplete(function(){

console.log("Running finally")

});

If you go to '/chain', you will get the following messages: chain1 chain2 Running onComplete

If you go to /link, you will get the following messages: link1 link2 Running onComplete

It really comes to play when doing asynchronous: app.get('/chain',function(context,next){ $.get('/my/url',function(){

console.log('chain2');

next(); });
